1. CVE Vulnerabilities   centos

  • Ensure your system's kernel version aligns with lxplus (CentOS 7.9) if applicable.

1.1. What is CVE?

  • CVE (Common Vulnerability & Exposure) is a publicly available catalog of security vulnerabilities.
  • Each CVE entry includes:
    • A unique ID for the vulnerability.
    • Detailed information about the vulnerability.
    • References to publicly available reports and patches.

1.2. How to Check if a CVE is Fixed

  • If you know the CVE number, you can verify whether the vulnerability has been addressed on your system.

1.2.1. Using `rpm` to Check CVE Fixes

  • Command to check if a package has addressed a specific CVE: 

    rpm -q --changelog [package-name] | grep [CVE-NUMBER]

  • Example for `openssl`: 

    rpm -q --changelog openssl | grep CVE-2021-3450
  • Result:
    • If the CVE number is not listed in the output, the vulnerability is not fixed on your system.

    • To fix, update the relevant package: 

      yum update openssl

1.2.2. Using `yum-plugin-security`

  • Install the `yum-plugin-security` package to enhance CVE management: 

    yum install yum-plugin-security

  • Check if a CVE has been addressed: 

    yum updateinfo info --cve CVE-2021-3445
  • Result:
    • No output: The CVE patch is already applied.
    • Output displayed: The system is still vulnerable.

  • Check for critical security updates: 

    yum --security --sec-severity=Critical check-update
