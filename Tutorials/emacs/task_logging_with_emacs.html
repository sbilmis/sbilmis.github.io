1. Introduction

This guide explains how to organize your daily task logs using a task-based structure and track your time efficiently using clocking and Org Capture in Emacs Org mode.

This approach is ideal for people who:

  • Work on multiple ongoing topics/projects
  • Want to keep logs in a single Org file
  • Need to track how much time is spent per topic

You can store your logs in any Org file, such as: `~/org/dailylog.org`

2. Step-by-Step Setup

2.1. 1. Organize Your Org File by Task

Create an Org file with top-level headings for each recurring topic of work. Example:

#+BEGINEXAMPLE

3. Project1

4. Project2

5. Project3

#+ENDEXAMPLE

Each day, you will add a second-level heading (subheading) for the date:

#+BEGINEXAMPLE

5.1. [2025-04-06 Sun]

  • [ ] Tasks or notes go here

#+ENDEXAMPLE

This creates a clean, searchable log per topic and per day.

5.2. 2. Track Time with Clocking

Org Mode lets you track time per heading.

  • `C-c C-x C-i` → Clock in (start tracking time)
  • `C-c C-x C-o` → Clock out (stop tracking)

Clocking will automatically create a `:LOGBOOK:` drawer like this:

#+BEGINEXAMPLE

5.3. [2025-04-06 Sun]

  • [ ] Work done listed here

#+ENDEXAMPLE

You can clock in/out multiple times under the same heading. Org Mode will record all sessions.

7.1. 4. Auto-Capture Logs into the Correct Section

To avoid manually navigating the file every time, use `org-capture` to insert tasks directly under the correct topic and date.

First, define these helper functions in your Emacs config: 

(defun my/org-capture-goto-today-heading (headline)
  "Goto HEADLINE in file and find or create today's subheading."
  (let ((file "~/org/daily_log.org")
        (today (format-time-string "[%Y-%m-%d %a]")))
    (find-file file)
    (goto-char (point-min))
    (re-search-forward (format "^\\* %s" (regexp-quote headline)) nil t)
    (unless (re-search-forward (format "^\\*\\* %s" (regexp-quote today)) nil t)
      (re-search-forward (format "^\\* %s" (regexp-quote headline)) nil t)
      (org-end-of-subtree t t)
      (insert (format "\n** %s\n" today)))
    (goto-char (point-at-eol))
    (org-end-of-subtree t t)))

(defun my/org-capture-into-today (headline)
  (my/org-capture-goto-today-heading headline)
  (set-buffer (current-buffer))
  (goto-char (point-max)))

Then, define your `org-capture-templates`: 

(setq org-capture-templates
      '(("1" "Project1" plain
         (function (lambda () (my/org-capture-into-today "Project1")))
         "- [ ] %?\n" :empty-lines 1)

        ("2" "Project2" plain
         (function (lambda () (my/org-capture-into-today "Project2")))
         "- [ ] %?\n" :empty-lines 1)

        ("3" "Project3" plain
         (function (lambda () (my/org-capture-into-today "Project3")))
         "- [ ] %?\n" :empty-lines 1)))

You can add as many templates as you like for additional projects.

7.2. Capture Usage

Use `C-c c 1` to quickly log something under Project1, `C-c c 2` under Project2, and so on.

Each capture:

  • Goes to the correct topic heading
  • Finds or creates today's `** [YYYY-MM-DD]` heading
  • Inserts a `- [ ]` task item to fill in

7.3. 5. View Open Checkbox Tasks with org-agenda

If you want to view all `[ ]` checkbox items across your log file:

  1. Press `C-c a` to open the agenda menu.
  2. Press `/` to filter.
  3. Type `[ ]` and press `RET` to search for all open checkboxes.

You can also define a custom command in your config: 

(setq org-agenda-custom-commands
      '(("c" "Open Checkbox Tasks"
         search "[ ]"
         ((org-agenda-files '("~/org/daily_log.org"))))))

Now `C-c a c` will show all your open checkbox tasks in the log.

7.4. Bonus Tips

  • Use tags like `:urgent:` or `:idea:` for filtering and searching
  • Store useful links, notes, and shell commands under daily headings
  • Export weekly summaries using `org-export`

8. Summary

This setup works great for personal logs, system admin tasks, research documentation, project tracking, and more.
