Funding — partial, from structured grant links only
Direct funding information is not available from the OpenAIRE Graph. OpenAIRE does not index funding-acknowledgment text, so it cannot tell you which papers credit TRUBA, nor return a complete funding profile for a paper. What it does expose is the sparse set of structured grant links that publishers or repositories happened to deposit — present for only 150 of 2,268 papers (7%). The complete “who acknowledged TRUBA” picture is available only from Scopus's funding-text index (see Why the corpus is defined by Scopus, below). Read the figures here as an illustrative lower bound, not a funding census.

Why the corpus is defined by Scopus
We tested whether OpenAIRE could find the TRUBA papers on its own, and it cannot — for a structural reason: OpenAIRE indexes titles, abstracts and keywords, but not the funding-acknowledgment text, which is the one place “we used TRUBA” is written. Two ways to measure the failure, both from live API queries:
  • Recall ≈ 1.3%. A keyword search for TRUBA restricted to Turkey returns ~41 articles with DOIs; only 30 of them are in our 2,384-paper corpus. So even tolerating all the noise, a keyword search surfaces barely 1 in 75 of the papers that actually acknowledge TRUBA.
  • Precision ≈ 8%. Conversely, most of what the keyword search does return is not ours — “truba” means pipe or trumpet in several Slavic and Turkic languages (fisheries, aquaculture, a saxophone-and-trumpet study…).
A ULAKBIM keyword search returns 10,000+ — but those are papers collected from ULAKBİM's DergiPark/TR-Dizin repositories, not papers that acknowledge TRUBA computing. And OpenAIRE's structured funder=TUBITAK link (13,393 papers) describes a different population — TÜBİTAK-funded research, not TRUBA-using research — with only ~7% of our corpus carrying any linked grant at all. So the acknowledgment corpus can only be built from Scopus's funding-text index; OpenAIRE then enriches and cross-checks it. That asymmetry is itself a finding: for acknowledgment-based infrastructure monitoring, Scopus's indexed funding text is currently irreplaceable.
Methodology
ⓘ How the comparison is built
The corpus is defined by Scopus (papers whose funding text credits TRUBA — OpenAIRE cannot search acknowledgment text, so it cannot define this corpus independently). Each article DOI is looked up in the OpenAIRE Graph (legacy search API, batches of 50, no auth); 2,268 of 2,291 (99%) are found — that match rate is the coverage comparison. For the matched papers we compare, per year: record counts (coverage), OA share (Scopus native flag vs OpenAIRE bestaccessright = OPEN), and total citations (Scopus vs OpenAIRE BIP!). "Green" means a repository copy exists — the main reason OpenAIRE's OA share exceeds Scopus's. Funder links are a lower bound: OpenAIRE only carries grants that publishers/repositories deposited (~7% of papers here). SJR quartiles, and institution/ researcher leaderboards, are not reproduced from OpenAIRE — see "what each database is best for" above.
